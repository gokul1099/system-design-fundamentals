What is Quorum

⏱ 24 min read 🟡 Intermediate Quorum Consensus Distributed Systems

Core Concept

Background

In Distributed Systems, data is replicated across multiple servers for fault tolerance and high availability. Once a system decides to maintain multiple copies of data, another problem arises: how to make sure that all replicas are consistent, i.e., if they all have the latest copy of the data and that all clients see the same view of the data?

Solution

In a distributed environment, a quorum is the minimum number of servers on which a distributed operation needs to be performed successfully before declaring the operation's overall success.

Suppose a database is replicated on five machines. In that case, quorum refers to the minimum number of machines that perform the same action (commit or abort) for a given transaction in order to decide the final operation for that transaction. So, in a set of 5 machines, three machines form the majority quorum, and if they agree, we will commit that operation. Quorum enforces the consistency requirement needed for distributed operations.

In systems with multiple replicas, there is a possibility that the user reads inconsistent data. For example, when there are three replicas, R1, R2, and R3 in a cluster, and a user writes value v1 to replica R1. Then another user reads from replica R2 or R3 which are still behind R1 and thus will not have the value v1, so the second user will not get the consistent state of data.

What value should we choose for a quorum? More than half of the number of nodes in the cluster: where is the total number of nodes in the cluster, for example:

  • In a 5-node cluster, three nodes must be online to have a majority.
  • In a 4-node cluster, three nodes must be online to have a majority.
  • With 5-node, the system can afford two node failures, whereas, with 4-node, it can afford only one node failure. Because of this logic, it is recommended to always have an odd number of total nodes in the cluster.

Beginner Analogy

💡
Quorum is like a committee vote where a decision is only valid if more than half the members agree. In a five-node distributed database, a quorum write means at least three nodes must acknowledge storing the data before the write is considered successful. This guarantees that any subsequent quorum read — also requiring three nodes to respond — will always overlap with at least one node that has the latest write. It's the distributed systems equivalent of 'if more than half agree, the truth cannot be split across two contradicting groups.'

Real-World Examples

💡 Cassandra (Facebook origin)
Cassandra's consistency levels QUORUM (majority) and ALL (all nodes) implement quorum reads and writes. A common production configuration uses QUORUM reads and QUORUM writes, ensuring that in a 5-node cluster, any write acknowledged by 3 nodes will always be visible to any 3-node read — providing strong consistency despite no single master.
🗄️ Amazon
DynamoDB's internal replication across Availability Zones uses quorum-based consensus to ensure writes are durable before acknowledging success to the client. A write must be confirmed by a majority of replicas, ensuring that even if one AZ fails, the quorum of remaining replicas has the complete data.
🐙 GitHub
Zookeeper, used by Kafka and HBase (both critical to GitHub's infrastructure), uses Zab consensus protocol which is quorum-based. Zookeeper requires a majority of nodes to be available to elect a leader and process writes — a 5-node Zookeeper ensemble can tolerate 2 simultaneous node failures while maintaining quorum.

Interview Tips

  • 💡 Know the quorum formula: W + R > N guarantees that any read quorum will overlap with any write quorum, ensuring at least one node has the latest write.
  • 💡 Show the consistency vs. availability tradeoff with quorum: increasing W and R improves consistency but reduces availability (more nodes must be up); decreasing them improves availability but risks stale reads.
  • 💡 Mention that R=1, W=N achieves strong consistency with fast reads but makes writes require all nodes to be available — a fragile CP configuration.
  • 💡 Explain that W=1, R=N is the reverse: fast writes but slow reads requiring all nodes — used in write-heavy workloads where reads can be slow.

Key Takeaways

✅ Quorum is the minimum number of distributed nodes that must agree on an operation for it to be considered valid.
✅ The strong consistency condition is W + R > N, ensuring write and read quorums always overlap by at least one node.
✅ Quorum provides a tunable tradeoff: higher W and R values increase consistency at the cost of availability; lower values increase availability at the cost of consistency.
✅ Cassandra's QUORUM consistency level implements the W + R > N guarantee across its ring topology.
✅ Zookeeper and Raft consensus protocols use quorum-based voting to elect leaders and ensure that committed decisions survive any minority of node failures.
