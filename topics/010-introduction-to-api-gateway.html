Introduction to API Gateway

⏱ 4 min read 🟢 Beginner API Gateway Microservices

Core Concept

An API Gateway is a server-side architectural component in a software system that acts as an intermediary between clients (such as web browsers, mobile apps, or other services) and backend services, microservices, or APIs.

Its main purpose is to provide a single entry point for external consumers to access the services and functionalities of the backend system. It receives client requests, forwards them to the appropriate microservice, and then returns the server’s response to the client.

The API gateway is responsible for tasks such as routing, authentication, and rate limiting. This enables microservices to focus on their individual tasks and improves the overall performance and scalability of the system.

Beginner Analogy

💡
An API Gateway is like the concierge desk at a large hotel. Every guest (client) goes to the concierge desk first, regardless of what they need — restaurant reservations, taxi bookings, or room service. The concierge authenticates them, figures out which department handles their request, and routes them appropriately, acting as a single, smart entry point.

Real-World Examples

🎬 Netflix
Netflix uses Zuul as their API Gateway to handle authentication, routing, and dynamic routing for over 1000 device types including TVs, phones, and browsers. Zuul shields 500+ backend microservices from direct client exposure.
📦 Amazon
AWS API Gateway is a fully managed service used by millions of applications to expose REST, HTTP, and WebSocket APIs. Amazon uses it internally to provide a consistent entry point for their own microservice architectures.
🚗 Uber
Uber built a custom API Gateway that handles authentication, rate limiting, and request routing for their mobile and web clients. It routes to different backend services based on the endpoint path and handles versioning of APIs.

Interview Tips

  • 💡 Describe the API Gateway as the single entry point for all clients, emphasizing cross-cutting concerns: auth, rate limiting, logging, and routing.
  • 💡 Distinguish an API Gateway from a simple load balancer — the gateway understands application semantics (routes by URL path, inspects JWT tokens) while a load balancer routes by connection-level metrics.
  • 💡 Mention that an API Gateway can become a bottleneck and SPOF — it must be horizontally scalable and highly available itself.

Key Takeaways

✅ An API Gateway is a server that acts as the single entry point for all client requests to a microservices backend.
✅ It centralizes cross-cutting concerns: authentication, authorization, rate limiting, SSL termination, and logging.
✅ The gateway decouples clients from the internal service topology, allowing services to be refactored without changing client code.
✅ Popular implementations include AWS API Gateway, Kong, NGINX, and Netflix Zuul.
