Introduction to Load Balancing

⏱ 4 min read 🟢 Beginner Load Balancing High Availability

Core Concept

Load balancing is a crucial component of System Design, as it helps distribute incoming requests and traffic evenly across multiple servers. The main goal of load balancing is to ensure high availability, reliability, and performance by avoiding overloading a single server and avoiding downtime.

Typically a load balancer sits between the client and the server accepting incoming network and application traffic and distributing the traffic across multiple backend servers using various algorithms. By balancing application requests across multiple servers, a load balancer reduces the load on individual servers and prevents any one server from becoming a single point of failure, thus improving overall application availability and responsiveness.

Beginner Analogy

💡
Imagine a busy supermarket with only one checkout lane — everyone waits in a single long queue. Load balancing is like opening multiple checkout lanes and having a greeter direct each shopper to the shortest line. No single cashier gets overwhelmed, and customers get served much faster.

Real-World Examples

☁️ Cloudflare
Cloudflare uses load balancing to distribute billions of DNS queries and HTTP requests across its global network of data centers. When one data center becomes congested, traffic is automatically shifted to healthier nodes.
🐦 Twitter/X
Twitter handles viral tweet storms that can spike traffic by 10x in seconds. Load balancers distribute incoming API requests across fleets of servers so that no single machine becomes the bottleneck during trending events.
🐙 GitHub
GitHub serves millions of git operations, web requests, and CI/CD webhook events simultaneously. Load balancers ensure that repository clones, pull request views, and Actions jobs are spread evenly across server pools.

Interview Tips

  • 💡 When asked about load balancing, distinguish between L4 (transport layer) and L7 (application layer) load balancers — the choice affects routing capabilities and performance overhead.
  • 💡 Always mention health checks: a load balancer without health checking can route traffic to failed servers, making it a single point of failure itself.
  • 💡 Discuss session persistence (sticky sessions) as a trade-off: it simplifies stateful applications but reduces the effectiveness of load distribution.
  • 💡 Bring up the load balancer itself as a potential SPOF and how to address this with active-passive or active-active HA pairs.

Key Takeaways

✅ Load balancing distributes incoming traffic across multiple servers to prevent any single server from becoming overwhelmed.
✅ It improves both availability and performance by routing around failed or slow servers.
✅ Load balancers perform continuous health checks to detect and remove unhealthy servers from the pool.
✅ A load balancer can operate at Layer 4 (TCP/UDP) or Layer 7 (HTTP) of the OSI model.
