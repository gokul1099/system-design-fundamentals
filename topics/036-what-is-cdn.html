What is CDN

⏱ 4 min read 🟢 Beginner CDN Performance Global

Core Concept

A Content Delivery Network (CDN) is a distributed network of servers strategically located across various geographical locations to deliver web content, such as images, videos, and other static assets, more efficiently to users. The primary purpose of a CDN is to reduce latency and improve the overall performance of web applications by serving content from the server nearest to the user. CDNs can also help improve reliability, availability, and security of web applications.

How CDNs work?

When a user requests content from a web application, the request is routed to the nearest CDN server (also known as an edge server) based on factors such as network latency and server load. The edge server then checks if the requested content is already cached. If it is, the content is served directly from the cache; otherwise, the edge server fetches the content from the origin server, caches it, and serves it to the user. Subsequent requests for the same content can then be served from the cache, reducing latency and offloading traffic from the origin server.

Beginner Analogy

💡
A CDN (Content Delivery Network) is like a chain of local convenience stores stocked with the most popular items from a central warehouse far away. If you need milk, you don't drive to the main distribution center — you walk to the store on your corner. The convenience store doesn't have everything, but it has the things most people need, and it gets there in seconds instead of hours.

Real-World Examples

🎬 Netflix
Netflix built its own CDN called Open Connect, with appliances installed directly inside ISP networks worldwide. Popular TV episodes are pre-positioned on these appliances before peak viewing hours. This means most Netflix traffic never traverses the public internet — it flows directly from ISP-hosted servers to the viewer.
☁️ Cloudflare
Cloudflare operates one of the world's largest CDN networks with over 300 points of presence in more than 100 countries. Static assets, images, scripts, and cached API responses are served from the nearest Cloudflare edge. This often cuts latency from hundreds of milliseconds to under 20ms for end users.
🛒 Amazon
Amazon CloudFront is Amazon's CDN service used by e-commerce, AWS customers, and Amazon Prime Video. Product images, JavaScript bundles, and video streams are cached at CloudFront edge locations. Dynamic API requests can also benefit from CloudFront via Lambda@Edge for personalization at the edge.

Interview Tips

  • 💡 Define CDN concisely: a geographically distributed network of servers that caches and delivers content from locations close to end users.
  • 💡 Explain the two main benefits: reduced latency (content served from nearby edge nodes) and reduced origin load (fewer requests reach the origin server).
  • 💡 Mention that CDNs are not just for static files — modern CDNs cache API responses, handle SSL termination, provide DDoS protection, and run edge compute.
  • 💡 Discuss CDN trade-offs: content may be slightly stale (TTL-based), and origin-fetching costs can increase if cache hit rates are low.

Key Takeaways

✅ A CDN is a distributed network of servers that caches and delivers content from locations geographically close to users.
✅ CDNs reduce latency by serving cached content from edge nodes rather than a distant origin server.
✅ CDNs reduce origin load by absorbing a large percentage of requests at the edge before they reach the origin.
✅ Modern CDNs provide additional services including DDoS mitigation, SSL termination, and edge computing.
✅ Major CDN providers include Cloudflare, Amazon CloudFront, Akamai, and Fastly.
