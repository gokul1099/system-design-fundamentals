URL vs URI vs URN

⏱ 4 min read 🟢 Beginner URL URI Networking

Core Concept

Understanding the differences between URL, URI, and URN can be crucial in web development and networking. Let's break down these concepts:

URL (Uniform Resource Locator):

  • Definition: A URL is a specific type of URI that not only identifies a resource on the internet but also provides a method to locate it by describing its primary access mechanism, usually its network location.
  • Components: It typically includes a protocol (such as HTTP, HTTPS, FTP), domain name, and path, optionally followed by query parameters or a fragment identifier.
  • Example: https://www.example.com/path?query=term#section
  • Key Characteristics:
    • Specifies how the resource can be accessed (protocol).
    • Includes the location of the resource (like a web address).

URI (Uniform Resource Identifier):

  • Definition: A URI is a generic term used to identify a resource either by location, name, or both. It serves as a universal identifier for resources on the internet.
  • Scope: All URLs and URNs are URIs, but not all URIs are URLs or URNs.
  • Example: A URL https://www.example.com is also a URI, and a URN like urn:isbn:0451450523 (identifying a book by its ISBN) is also a URI.
  • Key Characteristics:
    • A more general concept than both URL and URN.
    • It can be either a locator (URL), a name (URN), or both.

URN (Uniform Resource Name):

  • Definition: A URN is a type of URI that names a resource without describing how to locate it. It’s used to assign a unique and persistent identifier to a resource.
  • Example: urn:isbn:0451450523 uniquely identifies a book using its ISBN, irrespective of where it exists.
  • Key Characteristics:
    • Provides a unique and persistent identifier.
    • Does not specify a location or method to access the resource.

Summary of Differences:

  • URL: Specifies both the identity and the location of a resource (How and Where).
  • URI: A more comprehensive term covering both URLs (identifying and locating) and URNs (just identifying).
  • URN: Focuses only on uniquely identifying a resource, not on where it is located or how to access it.

In practical terms, when you’re browsing the internet, you're mostly dealing with URLs. URIs and URNs come more into play in specific contexts like software development, digital libraries, and systems where unique and persistent identification of a resource is crucial.

Beginner Analogy

💡
Think of a library catalog system. A URL is like the full shelf location: 'Floor 2, Aisle 3, Shelf B, Book 42 (available at the Main Library on Oak Street)'. A URN is just the book's unique ISBN — it identifies the book forever regardless of where it is stored. A URI is the general concept that covers both shelf locations and ISBNs — any string that identifies a resource.

Real-World Examples

🔍 Google
Google's search URLs are technically URIs: 'https://www.google.com/search?q=system+design' is a URL (a locator) because it specifies both the protocol, host, path, and query parameters needed to locate and retrieve a specific search result page.
📦 Amazon
Amazon uses URNs to identify digital books in their Kindle ecosystem: an Amazon Standard Identification Number (ASIN) acts as a URN — it permanently identifies a specific edition of a book regardless of which server currently hosts the file.
🐙 GitHub
GitHub repository URLs are URIs that are also URLs: 'https://github.com/user/repo' specifies the protocol, host, and path. These URLs are stable enough to serve as permanent identifiers for open-source projects referenced in academic papers and documentation.

Interview Tips

  • 💡 State the hierarchy clearly: URI is the superset; URL and URN are subtypes. Every URL is a URI, but not every URI is a URL.
  • 💡 A URL includes location/access information (protocol + host); a URN provides a persistent name without specifying where to find the resource.
  • 💡 In practice, 'URL' is the term used in everyday engineering conversations; knowing the formal URI/URN distinction demonstrates attention to specification detail.

Key Takeaways

✅ URI (Uniform Resource Identifier) is the broadest concept — any string that identifies a resource.
✅ URL (Uniform Resource Locator) is a URI that also specifies how to locate the resource (protocol, host, path).
✅ URN (Uniform Resource Name) is a URI that provides a persistent, location-independent identifier (e.g., ISBN, UUID).
✅ All URLs are URIs, but not all URIs are URLs.
✅ In practice, 'URL' is used colloquially even when 'URI' is technically correct.
