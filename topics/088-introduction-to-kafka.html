Introduction to Kafka

⏱ 4 min read 🟡 Intermediate Kafka Messaging Streaming

Core Concept

What is Kafka?

Apache Kafka is an open-source publish-subscribe-based messaging system. It is distributed, durable, fault-tolerant, and highly scalable by design. Fundamentally, it is a system that takes streams of messages from applications known as producers, stores them reliably on a central cluster (containing a set of brokers), and allows those messages to be received by applications (known as consumers) that process the messages.

Beginner Analogy

💡
Kafka is like a newspaper printing press combined with a library. Producers are reporters who submit articles (events) to the press. Kafka prints and stores every issue permanently. Consumers are readers who can pick up any issue — today's or one from six months ago — at their own pace. Multiple readers can read the same issue independently.

Real-World Examples

💼 LinkedIn
LinkedIn created Kafka to handle 1 trillion+ events per day — profile views, InMail sends, connection activity. Kafka's log-based storage lets multiple downstream systems (feed, notifications, analytics) each replay the same event stream independently.
🚗 Uber
Uber uses Kafka as the backbone of its real-time data pipeline — driver locations, trip events, and surge pricing signals flow through Kafka topics. uReplicator (open-sourced) replicates Kafka across data centers.
🎬 Netflix
Netflix uses Kafka for real-time monitoring and event streaming. Keystone Pipeline ingests billions of events per day from all Netflix services into Kafka, then routes to Elasticsearch, Cassandra, and Spark for analytics and alerting.

Interview Tips

  • 💡 Explain that Kafka is a log, not a queue — messages are retained and can be replayed, unlike traditional queues that delete on consumption.
  • 💡 Consumer groups enable parallel consumption: multiple instances in a group each own a partition subset.
  • 💡 Kafka ordering guarantee: within a partition, messages are strictly ordered. Across partitions, there's no ordering guarantee.
  • 💡 Use the partition key to route related events to the same partition (e.g., user_id ensures all events for a user are ordered).

Key Takeaways

✅ Kafka is a distributed, durable, append-only log — not a traditional queue.
✅ Topics are split into partitions, enabling parallel consumption across a consumer group.
✅ Messages are retained for a configurable period (not deleted on consumption) — enabling replay.
✅ Kafka provides high throughput (millions of messages/second) and low latency.
